Comparison of Git Repository Management

2018-07-05

There are tons of implementations of Git repository manager.

These two of them are gaining more and more attentions by developers:

  • GitLab

  • Gogs

Products GitLab Gogs
PROS
1 Very good Web UI Self-hosted, easy installation
2 Maintain by a professional team, regular updates Light weight, benefit from GoLang
3 Multi severities of branch, eg: protected branch, etc Open Source(Well… CONS:2)
4 Built-in CI/CD -
5 Group Milestone -
6 Powerfull issue(s) tracker Simple issue(s) tracker
7 Built-in container registry -
CONS
1 Suffer from installation and configuration(s) relatively Still young, not as much functionalities as GitLab
2 Consume relatively massive resources at runtime Maintain by one and only one developer(may be a pit..)
3 Upgrade process fails more often than not Not too much plugins or Webhooks
4 Sometimes raises error while backups(due to resources consumption) Can not make pull requests between branches of forked repos
5 No code review supported No code review supported

My points of view

It doesn’t truelly matter which Git repository manager that you choose,

but it does matter how you use Git and how to use it properly.

Rationally

  • It has been over 8 years since GitLab published its first release which means that it’s supreme stable while Gogs is maintained by one developer which may bring us unstable circumstances.

  • About CI/CD. Both git repo manager can be integrated with Jenkins. There are not bunch of works. However, GitLab proivdes a natively build-in CI/CD which will be a convenient way to achieve.

  • Seems like, GitLab is a super set of Gogs, though it’s not that correct in some point of views. I think Gogs would be a good choise a adventurer who wants to give a shot for Git or a very small team who keep scm simple and quickly. However, if your are a company who keeps a bunchs of projects and teams working respectively, I think GitLab would be a better choice.

Emotionally

  • Go is a wonderful language and I really appreciate that Gogs is implemented by it. But it is still too yound and lacks of functionalities.

  • We have been using GitLab since 2014 and everything seems wonderful.

    • Web UI
    • Administration
    • Full docs
    • Community support
